## 决策支持系统（DSS）发展阶段对比表

| 阶段               | 时间             | 核心驱动    | 关键技术 / 代表系统                         | 主要能力        | 决策特点              |
| ---------------- | -------------- | ------- | ----------------------------------- | ----------- | ----------------- |
| **萌芽阶段**         | 1960s–1970s    | 模型驱动    | 运筹学、线性规划、排队模型                       | 方案计算、定量分析   | **帮人算**：计算结果供专家决策 |
| **数据驱动 DSS**     | 1980s          | 数据驱动    | RDBMS、SQL、报表系统                      | 历史数据查询、趋势分析 | **帮人看**：基于事实回顾    |
| **DSS 与 MIS 融合** | 1990s          | 企业信息化   | MIS、ERP                             | 企业级数据整合     | 面向中高层管理决策         |
| **数据仓库 & OLAP**  | 1990s 后期–2000s | 数仓驱动    | Data Warehouse、ETL、OLAP（Hive/Spark） | 多维分析、聚合、钻取  | 多角度、体系化分析         |
| **BI 阶段**        | 2000s–2010s    | 自助分析    | Tableau、Power BI、BO                 | 可视化、即席分析    | **业务自助决策**        |
| **智能 DSS（IDSS）** | 2010s–至今       | 数据 + 智能 | 机器学习、实时计算、规则引擎                      | 预测、推荐、异常检测  | **系统参与判断**        |
| **AI 决策自动化**     | 近几年            | AI 驱动   | 大模型、NLP、Agent                       | 自动决策、闭环执行   | **帮人做** / 半自动决策   |

## SDLC vs CLDS

### 1、CLDS 与 SDLC 基本定义对比表

| 项目     | SDLC                                    | CLDS                                                                |
| ------ | --------------------------------------- | ------------------------------------------------------------------- |
| 全称     | Software Development Life Cycle         | Common Lifecycle for Data Science                                   |
| 中文     | 软件开发生命周期                                | 数据科学通用生命周期                                                          |
| 定义     | 一个软件系统从需求到下线的完整开发过程                     | 专门为 **数据分析 / 数据科学 / 机器学习 / 智能决策** 设计的一套生命周期方法论                      |
| 面向对象   | 软件系统                                    | 数据 / 模型 / 决策                                                        |
| 解决核心问题 | **系统怎么做**                               | **数据能说明什么**                                                         |
| 输入     | 明确的需求                                   | 不确定的数据                                                              |
| 输出     | 功能正确的软件                                 | 指标、分析结论、模型                                                          |
| 成功标准   | 程序可运行、稳定、可维护、按时交付                       | 业务效果是否达成、模型可信、结论稳定、持续优化                                             |
| 生命周期形态 | 瀑布式，偏线性<br/>SDLC：需求 → 设计 → 开发 → 测试 → 上线 | 螺旋式，高度迭代<br/>CLDS：业务问题<br/>   ↘ 数据 ↘ 分析 ↘ 建模<br/>       ↖ ← 评估 ← 监控 |

---

### 2、生命周期阶段对比表（重点）

| 阶段顺序 | SDLC（软件开发） | CLDS（数据科学）  |
| ---- | ---------- | ----------- |
| 1    | 需求分析       | 业务理解        |
| 2    | 系统设计       | 数据获取        |
| 3    | 开发实现       | 数据理解（EDA）   |
| 4    | 测试         | 数据准备 / 特征工程 |
| 5    | 部署         | 建模          |
| 6    | 运维         | 评估          |
| 7    | 迭代         | 部署          |
| 8    | —          | 监控与持续优化     |

✅ **关键差异**

- SDLC：阶段相对固定
- CLDS：阶段经常来回反复

---

### 3、确定性 vs 不确定性对比表

| 对比角度   | SDLC  | CLDS    |
| ------ | ----- | ------- |
| 需求稳定性  | 相对稳定  | 经常变化    |
| 数据质量   | 不关心   | 高度依赖    |
| 过程可预测性 | 强     | 弱       |
| 是否保证成功 | 基本可保证 | 不一定成功   |
| 工作方式   | 按计划执行 | 探索 + 验证 |

- **SDLC 是“工程问题”**
- **CLDS 是“探索问题”**
